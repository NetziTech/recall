# Why this directory has no repositories

The retrieval domain models *ephemeral* outputs:

- `ContextBundle` is the result of `mem.context` and is consumed
  immediately by the JSON-RPC adapter. It is never stored and never
  reloaded.
- `RecallResult` is the output of `mem.recall` and is consumed
  immediately by the same adapter. It is never stored.

Persistence in the system already lives in the `memory` bounded
context: `DecisionRepository`, `LearningRepository`, `EntityRepository`,
`TaskRepository`, `TurnRepository`, `SessionRepository`. The retrieval
pipeline composes their outputs through the `LexicalSearch` and
`VectorSearch` ports (which are NOT repositories: they return
projections, not aggregates).

If a future requirement asked the retrieval module to record the
bundles it produced for audit or replay (e.g. "show me the bundle that
was assembled at 14:32"), a `BundleRepository` interface would land
here. Until then, the directory remains empty by design.

The audit trail of what happened during a recall is captured by the
domain events (`ContextBundleAssembled`, `ContextLayerAdded`,
`ContextBundleTruncated`, `RecallExecuted`); subscribers in the
infrastructure (audit-log adapter, telemetry adapter) persist the
events themselves, not the bundles they describe.
